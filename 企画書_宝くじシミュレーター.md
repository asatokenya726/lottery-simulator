# 宝くじシミュレーター 1枚企画書（叩き台）

## 前提条件

| 項目 | 内容 |
|------|------|
| プラットフォーム | Webアプリ（ブラウザ）・レスポンシブ対応 |
| ターゲット市場 | 日本国内のみ |
| 開発体制 | 個人開発（1人）・予算ほぼゼロ |

---

## 1) 課題（現状・損失）

| 観点 | 現状 |
|------|------|
| 宝くじ市場の縮小 | ピーク2005年の1兆円超→2023年は8,088億円へ約30%減少 |
| 若年層の疎外感 | 主力購入層は50〜70代（購入経験率86〜88%）。20〜30代はスマホゲーム・動画配信に可処分時間を奪われ、宝くじに触れる機会がない |
| 「損する」体験の不透明さ | 宝くじの期待値は約47%（還元率）。「100万円買ったら47万円返ってくる」を体感で理解できる場がない。買って損した後に後悔するしかない |
| 既存シミュレーターの限界 | 既存の高速宝くじシミュレーター等は「統計ツール」止まり。ゲーム性・継続動機・コミュニティ要素がゼロ |

---

## 2) ターゲット（一次/ユーザ属性）

| 区分 | 属性 |
|------|------|
| 一次ターゲット | 20〜35歳男性、スマホゲーム経験者、ガチャ文化に馴染みがある層 |
| ペルソナ | 「宝くじって当たるの？」と興味はあるが、わざわざ売り場に行く気はない。ソシャゲの10連ガチャのノリで気軽に試したい |
| 二次ターゲット | 宝くじ購入経験者（40〜60代）で「買い続けたらどうなるか」を検証したい層 |
| 除外 | ギャンブル依存症傾向の強い層（注意喚起UIを設置） |

---

## 3) 提供価値（Before → After）

| Before | After |
|--------|-------|
| 宝くじに興味はあるが、実際に買うのは面倒・もったいない | ゲーム内通貨でノーリスクで「10連ガチャ」感覚で宝くじ体験 |
| 当選確率がピンとこない（「1000万分の1」が体感できない） | 数百回〜数千回のシミュレーションを通じて「絶望的に当たらない」を体で理解 |
| 当たるかもという曖昧な期待で毎年数万円を消費 | 仮想体験で満足 or 逆に「本物を買いたくなる」教育的エンターテインメント |
| ソシャゲのガチャに課金しているが、宝くじの確率感覚がない | 宝くじの確率を「ガチャ文化のUI」で直感的に理解 |

---

## 4) 差別化（なぜ選ばれる）

| 競合 | 弱点 | 本サービスの差別化 |
|------|------|-------------------|
| 高速宝くじシミュレーター（Web） | 統計ツール。ゲーム性ゼロ、継続動機なし | 「所持金」「毎日補給」のゲームループで継続性を担保 |
| リアル宝くじシミュレーター（Android 10万DL, 評価4.5） | ネイティブアプリのみ。Web版なし。広告が多い | Webで即プレイ。インストール不要。10連ガチャUIで差別化 |
| Lotto Simulation（海外Web） | 海外宝くじのみ。日本の宝くじ非対応 | 日本の宝くじ（ジャンボ・ロト6・ロト7・スクラッチ）の実確率を完全再現 |
| ガチャメーカー等のガチャシム | 排出率をユーザーが設定する汎用ツール。宝くじ特化ではない | 現実の宝くじ確率をプリセット。「ガチャ×宝くじ」のニッチを独占 |
| Silver Games Lottery Simulator（海外Web） | 英語のみ。日本宝くじ非対応 | 日本語・日本宝くじ特化 |

コア差別化: 既存サービスは「統計ツール」か「単発シミュレーター」。「ゲーム内経済圏＋10連ガチャUI＋毎日ログインループ」を宝くじに組み合わせたサービスは存在しない。

## 5) 利用シーン（トリガー、頻度、環境）

| 項目 | 内容 |
|------|------|
| トリガー | (1) 年末ジャンボ等の宝くじシーズンのSNS話題 (2)「宝くじ 確率」でググった時 (3) 暇つぶし・通勤中 |
| 頻度 | 毎日1〜3回（0時の資金補給をトリガーに日課化を狙う） |
| 1セッション | 2〜5分（10連ガチャを数回回して結果確認） |
| 環境 | スマホブラウザ（80%想定）、PCブラウザ（20%想定） |
| 共有動機 | 「1等当たった！」スクリーンショットをX(Twitter)に投稿 → バイラル導線 |

## 6) MVP範囲（やる/やらない）

| やる（MVP） | やらない（将来版） |
|-------------|-------------------|
| ゲーム内通貨（初期10万円分） | リアルマネー課金 |
| 10連ガチャUI（宝くじ10枚購入） | ユーザー間フリマ/取引機能 |
| ジャンボ宝くじ1種類の確率再現 | 複数宝くじ種類（ロト6/7、スクラッチ等） |
| 当選演出（アニメーション/エフェクト） | ガチャ景品（コレクションアイテム） |
| 毎日0時の資金補給（上限付き） | ランキング/リーダーボード |
| 累計収支・当選履歴の表示 | ユーザーアカウント/ログイン機能 |
| SNSシェア機能（結果画像生成） | プッシュ通知 |
| レスポンシブデザイン（スマホ優先） | ネイティブアプリ化 |
| ギャンブル依存注意喚起の表示 | 多言語対応 |

MVP開発期間目安: 2〜4週間（個人開発、React or Next.js + LocalStorage）

## 8) 最大リスク

| リスク | 深刻度 | 対策 |
|--------|--------|------|
| 法規制リスク（富くじ罪） | 高 | ゲーム内通貨のみ使用、現金への換金不可を明確化。刑法187条の「富くじ」に該当しないよう、あくまで「シミュレーション/エンターテインメント」と明示。利用規約で「賭博行為ではない」旨を記載 |
| ギャンブル依存助長の批判 | 中 | ゲーム画面に「これはシミュレーションです」の常時表示。プレイ時間警告。依存症相談窓口へのリンク設置 |
| 継続率の低さ | 中 | 「資金が尽きる→0時に補給」のサイクルが弱い場合、ログインボーナス・連続プレイ報酬・累計実績バッジなどのリテンション施策を追加 |
| 「宝くじ」商標・著作権 | 低 | 「宝くじ」は一般名詞で商標にはならないが、公式ロゴ等の無断使用は回避。オリジナルデザインで対応 |
| フリマ機能の規制（将来） | 高 | RMT化した場合、資金決済法・景品表示法に抵触する可能性。将来のフリマ機能は法務相談必須 |

## 市場調査データ（参考）

### 宝くじ市場
- 2023年度売上: 8,088億円（ピーク2005年の1兆円超から約30%減）
- 宝くじ人口: 約5,005万人（成人の47.5%が年1回以上購入）
- 購入経験者: 約8,572万人（成人の81.4%）
- 還元率: 約46.8%（法律で上限50%と規定）
- 主力購入層: 50代（88.7%）、60代（86.9%）、40代（84.4%）

### 既存宝くじシミュレーター（主要サービス）
- 高速宝くじシミュレーター（japandailylife.com）: 2024年サマージャンボ想定、レトロゲーム風UI、速度調整機能あり
- リアル宝くじシミュレーター（Android）: 10万DL超、評価4.5、仕事で稼いだお金で宝くじを購入するゲーム
- 無限宝くじ7（iOS）: 過去実施の宝くじと同確率で無限に引ける
- 宝くじシミュレーション（unityroom）: Unityベースのスクラッチ式、2025年更新
- Lotto Simulation（海外）: Powerball/Mega Millions/EuroJackpot対応
- Silver Games Lottery Simulator: HTML5/WebGLブラウザゲーム

### ガチャシミュレーター市場
- ガチャメーカー（gacha.work）: オリジナルガチャ作成・共有サービス
- GameWith/Game8: ゲーム特化型ガチャシミュ（原神、ウマ娘、モンスト等）
- 汎用ガチャシミュレーターアプリ: 排出率カスタマイズ型
